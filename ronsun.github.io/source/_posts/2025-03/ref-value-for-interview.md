C# 的實值型別與參考型別雖然是入門主題，但真要深究是可以非常深入的且涵蓋很廣的，本文將嘗試透過面試的角度來一步一步由淺入深探討。

<!--more-->

### 實質型別和參考型別的差異
這個問題可以從幾個面向探討。  

#### 賦值 (assignment) 與比較 (equality) 時的差異
+ 實值型別的變數賦值時會完整複製實例 (instance) 的內容，比較時也是比較完整的內容。
+ 參考型別的變數賦值時是複製記憶體位址，比較時則是預設比較記憶體位址 (除非該參考型別有額外實作比較行為)。  
  - 因此實值型別的變數大小就是完整內容的尺寸；反之，參考型別的變數則是固定的大小。

#### 記憶體配置的差異
+ 實值型別**通常**都是儲存在堆疊 (stack) 中；參考型別則**一定**儲存在託管堆積 (managed heap) 中。
  - 實值型別在裝箱 (boxing) 過程被移動到堆積上；拆箱 (unboxing) 時則是移動回堆疊。
    * 裝箱與拆箱過程會增加執行的時間成本。
  - 實值型別如果是參考型別的成員，則初始化時就會和參考型別一起存在堆積中。
+ 根據 ChatGPT 的回答，上面的堆疊與堆積並不是資料結構上所說的 Stack 與 Heap，而是兩種不同的記憶體管理方式的名字，名稱只是借用了相關概念，而不是用相應的資料結構實作。

#### 生命週期的差異
+ 存在堆疊中的那些實值型別的實例的生命週期與它所在的作用域相同。
  - 作用域 (scope) 指的是由大括號 `{}` 所界定的程式碼區塊。
+ 參考型別則是存在堆積中，由 .NET 垃圾回收機制 (GC) 管理。

### 如何分辨實值型別和參考型別
+ 列舉 (enum) 和結構 (struct) 都實值型別，除此之外的所有型別都是參考型別。
